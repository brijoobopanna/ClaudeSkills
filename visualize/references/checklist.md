# Quality Checklist

Run through this before delivering any visual-brief output.

## Structure
- [ ] Layout pattern matches the content's primary story
- [ ] No more than 5-7 elements per horizontal row
- [ ] Sections are clearly separated with monospace headers
- [ ] Vertical flow reads top-to-bottom naturally
- [ ] Horizontal flow reads left-to-right naturally

## Visual hierarchy
- [ ] Scannable at a glance — structure visible before reading text
- [ ] ONE hero element with accent background (not zero, not three)
- [ ] Gates/reviews visually distinct from regular boxes
- [ ] Future/planned items use dashed borders, not solid
- [ ] Section headers stand out from box content

## Color
- [ ] Color strategy matches diagram complexity
- [ ] Colors encode meaning (function, trust level) not sequence
- [ ] Sufficient contrast — all text readable on its background
- [ ] Accent color used sparingly — hero + gate + callout, not everywhere

## Content
- [ ] Box titles: 1-3 words
- [ ] Box details: max 4 lines, each under 30 characters
- [ ] Annotations clearly secondary (italic, muted)
- [ ] Arrow labels explain the transformation ("compiles", "reviews")
- [ ] No jargon without context — annotations can clarify

## Connections
- [ ] Solid arrows for primary/forward flow
- [ ] Dashed arrows for secondary/rejection/feedback paths
- [ ] Arrows don't visually cross boxes they don't connect to
- [ ] Feedback loop is shown explicitly (not just implied)

## Completeness
- [ ] Author attribution at top (if visualizing someone's content)
- [ ] Core insight callout at bottom
- [ ] Compound loop strip shown (if system self-improves)
- [ ] "Looking ahead" section (if future plans mentioned in source)

## Technical
- [ ] Single HTML file
- [ ] Only Google Fonts as external dependency
- [ ] Responsive — works at 375px and 1200px widths
- [ ] All text minimum 11px
- [ ] No JavaScript required for static layouts
