# Quality Checklist

Run through before delivering any visual-brief output.

## Structure
- [ ] Layout pattern matches the content's primary story
- [ ] No more than 5-7 elements per horizontal row
- [ ] Sections clearly separated with monospace headers
- [ ] Flow reads naturally (left-to-right or top-to-bottom)

## Visual hierarchy
- [ ] Scannable at a glance before reading text
- [ ] ONE hero element with accent background
- [ ] Gates/reviews visually distinct from regular boxes
- [ ] Future/planned items use dashed borders

## Color
- [ ] Color strategy matches diagram complexity
- [ ] Colors encode meaning (function, trust, status) not sequence
- [ ] Sufficient contrast — all text readable
- [ ] Accent color used sparingly

## Content
- [ ] Box titles: 1-3 words
- [ ] Box details: max 4 lines, each under 30 characters
- [ ] Arrow labels explain the transformation
- [ ] Annotations clearly secondary (italic, muted)

## Connections
- [ ] Solid arrows for primary flow
- [ ] Dashed arrows for secondary/rejection/feedback paths
- [ ] Arrows don't cross unrelated boxes
- [ ] Feedback loop shown explicitly

## Completeness
- [ ] Author attribution at top (if visualizing someone's content)
- [ ] Core insight callout at bottom
- [ ] Compound loop strip (if system self-improves)

## Gap analysis specific
- [ ] "Have" and "missing" use different colors (teal vs coral)
- [ ] Gap boxes explain consequences, not just labels
- [ ] Strengths shown too — not purely negative
- [ ] Priority order shown if recommending build sequence

## Technical
- [ ] Single file, only Google Fonts as external dependency
- [ ] Responsive at 375px and 1200px widths
- [ ] All text minimum 11px
